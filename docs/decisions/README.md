# Decision Log (ADRs)

Settled architecture choices for Yes Chef. **Check these before proposing
architecture changes; don't re-litigate them.** General "how Jon builds software"
decisions live in `~/code/jon-platform`; these record how those bind to Yes Chef
plus any app-specific calls.

- [ADR-0001](ADR-0001-persistence-sqlitedata.md) — Persistence is SQLiteData (not SwiftData/Core Data)
- [ADR-0002](ADR-0002-cloudkit-sync-no-server.md) — Sync is CloudKit via SQLiteData; no server, no auth
- [ADR-0003](ADR-0003-private-libraries-recipe-transfer.md) — Private per-person libraries; recipes move by transfer (copy), not co-editing
- [ADR-0004](ADR-0004-structured-recipe-editor.md) — MVP editor uses text entry with structured, non-destructive persistence
- [ADR-0005](ADR-0005-image-storage-and-processing.md) — Recipe images use a pure processing pipeline and recipe-owned storage rows
- [ADR-0006](ADR-0006-taxonomy-source-and-library-placement.md) — Categories stay flexible; source, author, placement, and recipe families are typed concepts
- [ADR-0007](ADR-0007-web-recipe-capture-engine.md) — Web recipe capture harvests Galavant's parser now and converges on a shared package later
- [ADR-0008](ADR-0008-curated-collections.md) — Curated collections are a new sibling of Menu (editorial indexes); reserved now, built post-sync (Proposed)
- [ADR-0009](ADR-0009-in-app-authenticated-browser-capture.md) — In-app authenticated browser capture (rendered logged-in DOM) via WebExtractorKit; never store credentials
- [ADR-0010](ADR-0010-cloudkit-sync-enablement.md) — CloudKit sync enablement specifics: BLOB→CKAsset, lean original-provenance, upsert + dedup-on-read, clean cutover
- [ADR-0011](ADR-0011-actionable-chat-make-ahead.md) — Actionable chat (first cross-app instance): seeded recipe chat + typed `(extract → commit)` apply-action catalog; first verb make-ahead → new `Recipe.makeAhead` field; lifts `GalavantAI` → shared `LLMClientKit` (Accepted)
- [ADR-0012](ADR-0012-menu-actionable-chat.md) — Menu actionable chat: extends ADR-0011 to a composite subject (a whole menu); staged prep plan on `Menu` + complement verb → `MenuItem` + advisory chat; `Menu` scope first, planner-day later (Accepted; shipped PRs #81–#83)
- [ADR-0013](ADR-0013-meal-planner-actionable-chat.md) — Meal-Planner actionable chat: extends ADR-0012 to the absolute-date planner; day-scoped complement verb → `MealPlanItem` + advisory chat; no schema change (Accepted)

- [ADR-0022](ADR-0022-llm-aligned-compare-matrix.md) — LLM-aligned Compare matrix: semantic row alignment (chicken breast ≡ thigh; chile spellings; morita ≈ chipotle) + role ordering ("protein at top") for the Workbench Compare surface; boundary — LLM drives *presentational* alignment, grocery consolidation stays deterministic; structured-out/verbatim-cells, cached per candidate-set, deterministic `comparisonKey` fallback (Proposed)

- [ADR-0036](ADR-0036-promote-note-to-recipe.md) — Promote a **recipe-shaped note → a real recipe** (formalizes ADR-0027 D5/A6): parse note prose → `WorkbenchDraftRecipe` → the ADR-0024 editable-proposal review → committed `Recipe`, reusing web-capture extraction (`RecipeParseBuilder`/`IngredientParser`) and the existing review/commit surface; **rejects** the "just feed notes into grocery" alternative as a trap (secretly the same prose→ingredients parse, none of the reuse); the real new work is **placement + provenance** (note preserved, passive back-link, optional menu re-placement restoring the recipe-id invariant); S1 library commit, S2 menu re-placement (Accepted)
- [ADR-0040](ADR-0040-editable-at-the-grain-it-is-stored.md) — **LLM-populated content is human-editable at the grain it is stored**: if the human should be able to add/fix/delete one of a thing, that thing is a **row with an id**, never an element inside an encoded blob (the `Menu.prepPlan` BLOB is why the prep plan can only be *regenerated*, never repaired); **the human never authors the serialization format** — the `session:` / `- task → serves` DSL is a *transport*, and post-commit editing is structured fields + a session picker; any text we still parse is **lossless or loud**, never silently dropped (today's parser drops junk lines and re-derives `sourceDish` from task wording); prep-plan BLOB → `prepPlanSteps` rows **before the prod-schema cut locks it forever**; extends [[decompose-notes-into-typed-homes]], governs ADR-0034/0038 Amd 1/0039 (Accepted — S1+S2 shipped PR #184; S3 loud-parsing pass deferred)
- [ADR-0041](ADR-0041-playbook-section-toolbar-and-scoped-handoff.md) — **Playbook sections become self-serve units**: each section (Make-ahead, Chef It Up, Serve With) gets a **state-aware per-section toolbar** (external **Hand off** primary when empty, **Edit** when filled, **Clear** demoted to overflow) rendered in the expanded content so collapsed headers stay calm, its **own edit sheet** lifted off the monolithic recipe editor (reuses ADR-0024), and a **section-scoped external hand-off** — which requires `HandoffExportSource` to carry the section kind and the `matches()` router to key on it, **or two sections' pasted results cross-wire** (both are `.recipe` + same id today); ~~provenance + an optional live-chat `conversationURL` on a synced `PlaybookSectionMeta` sidecar~~ (**withdrawn, Amd 3 — never built**); **Notes is excluded** (heterogeneous residue; the `Learning` table is already the typed home per ADR-0038 Amd 1); **copy is brand-free** — verbs carry the path (Hand off vs Ask), "ChatGPT" lives once in Settings; **whole-recipe-from-Chat routes through ADR-0023** (adjust → overwrite/variation), not a section verb; S1 toolbar + edit-sheet, S2 the scoped hand-off + `matches()` fix, S2.5 non-destructive returns + collapsed toolbar (Amd 1), S2.6 destructive-action safety + Serve With swipe (Amd 2); lives in ADR-0039, governed by ADR-0040 (**COMPLETE — closed at S2.6, 2026-07-19, with no schema change of its own; S3 + `PlaybookSectionMeta` WITHDRAWN per Amd 3 — ChatGPT exposes no live `/c/` URL, and the meta-only fallback was rejected because the table's key was justified by the missing field and no provenance consumer exists**)

- [ADR-0042](ADR-0042-workbench-handoff-and-the-return-block.md) — **The workbench hands off; the return block is a terminal turn.** Extends ADR-0038 by adding `.workbench` as a hand-off source and by writing down the return-payload contract ADR-0038 left implicit. Three rules: **(D1)** the conversation is unconstrained, but the turn that emits `YC-HANDOFF:` behaves like a **function return, not an interlocutor** — every failure so far (the S2.5 blob-report: headings, nesting, "what the recipe already does well") was the model never being told to switch modes; **(D2)** *outboard deliberation, keep structured canonical writes in-app* — a bad advisory line costs one delete, a wrong ingredient identity silently corrupts a recipe and orphans ADR-0021 variation anchors, so `adjustRecipe` **stays an unwired placeholder**; **(D3)** prose is welcome exactly where it terminates in a human-read text field, and must be pinned wherever it feeds typed fields. **(D4)** the contract **leaves the payload entirely** and lives in the chat app's **project custom instructions** — generated from one Core constant, pasted via a Settings copy button, with a `YC-CONTRACT: v<n>` marker echoed in the return so a stale or missing paste fails *loudly* instead of silently drifting from the parser; one shared "Yes Chef" project to start, since a menu has a natural end and needs no per-source anchor. This also closes a real structural gap: `.immediate` prompts carry a literal format example, `.discuss` prompts carry **one sentence and no example** — which is the ADR-0041 S2.5 blob-report root cause, routed around but never fixed. Workbench deliverables: **compare** = named differences with a claim attached (never a per-recipe walkthrough — the matrix does that better), **experiments** = `{hypothesis, change, rationale}` → `workbenchLog` rows (**supersedes ADR-0019 S3's `experiments` BLOB** — the `Menu.prepPlan` defect again), **draft** deferred as a structured write. The log is how thinking returns now that there's no reopen link (D6). **Retires ADR-0023 S3's in-app refine loop** as unwanted, not unscheduled (D7). **S1 is schema-free** (`AIHandoff` is device-local; `workbenchLog` already exists); **S2 adds three nullable synced `workbenchLog` columns** — `hypothesis`/`change`/`rationale` — because an experiment is **write-many** (its `outcome` is filled in later) and ADR-0040 puts human-repairable content in typed fields, **not** smeared into `body` (OQ2, resolved 2026-07-20: pre-prod is the reason to fix the shape *now*, not a licence to defer it; contrast ADR-0041's withdrawn `PlaybookSectionMeta`, which failed on *justification*, not cost). **Hand-validated end-to-end (OQ6, 2026-07-20): all slices un-gated.** With the v1 contract living only in the project's custom instructions, a long real conversation switched cleanly into the return block on `finalize`, echoed `YC-CONTRACT: v1`, and held all three field labels — confirming D1/D4/D8 in the shape we'd ship. **One binding failure: blank lines between blocks did not survive the paste path**, so the parser splits on the **label cycle**, never whitespace. **(D9)** threads carry a derived `<TaskType>: <Object>` title — task, not object-kind, since ADR-0041 section-scoping gives one recipe three live threads — and the title is **advisory only** (the auto-titler paraphrased it in testing; nothing parses it). Also **D8: the experiments verb emits no learnings** — 3 of 4 returned "learnings" were its own untested hypotheses restated as fact, which is structural (a conjecture establishes nothing), so knowledge is recorded later in the experiment's `outcome` — an experiment is a *pending* learning (Accepted 2026-07-20; S0/S1 dispatchable, S2 the schema slice behind them)
- [ADR-0033](ADR-0033-recipe-detail-polish.md) — Recipe-detail polish (off a mockup): **reject** the magazine redesign (serif display title, second type family) — the shipping Reader already ships SF, bordered source buttons, and accent step badges; adopt only two things — **metadata chips** (hairline-bordered capsule treatment on the existing `recipeStats` labels) and a **servings-attached scaler** (the `Serves …` chip becomes a `Button` anchoring the `ScalePanel` popover + showing scaled servings; retire the toolbar scaler button — which currently **crashes** because its popover is anchored to a collapsed `.secondaryAction` menu item — and the duplicate summary); view-layer only, **no schema/model/sync/LLM** (Accepted)
