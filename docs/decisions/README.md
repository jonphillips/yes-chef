- [ADR-0042](ADR-0042-workbench-handoff-and-the-return-block.md) â€” **The workbench hands off; the return block is a terminal turn.** Extends ADR-0038 by adding `.workbench` as a hand-off source and by writing down the return-payload contract ADR-0038 left implicit. Three rules: **(D1)** the conversation is unconstrained, but the turn that emits `YC-HANDOFF:` behaves like a **function return, not an interlocutor** â€” every failure so far (the S2.5 blob-report: headings, nesting, "what the recipe already does well") was the model never being told to switch modes; **(D2)** *outboard deliberation, keep structured canonical writes in-app* â€” a bad advisory line costs one delete, a wrong ingredient identity silently corrupts a recipe and orphans ADR-0021 variation anchors, so `adjustRecipe` **stays an unwired placeholder**; **(D3)** prose is welcome exactly where it terminates in a human-read text field, and must be pinned wherever it feeds typed fields. **(D4)** the contract **leaves the payload entirely** and lives in the chat app's **project custom instructions** â€” generated from one Core constant, pasted via a Settings copy button, with a `YC-CONTRACT: v<n>` marker echoed in the return so a stale or missing paste fails *loudly* instead of silently drifting from the parser; one shared "Yes Chef" project to start, since a menu has a natural end and needs no per-source anchor. This also closes a real structural gap: `.immediate` prompts carry a literal format example, `.discuss` prompts carry **one sentence and no example** â€” which is the ADR-0041 S2.5 blob-report root cause, routed around but never fixed. Workbench deliverables: **compare** = named differences with a claim attached (never a per-recipe walkthrough â€” the matrix does that better), **experiments** = `{hypothesis, change, rationale}` â†’ `workbenchLog` rows (**supersedes ADR-0019 S3's `experiments` BLOB** â€” the `Menu.prepPlan` defect again), **draft** deferred as a structured write. The log is how thinking returns now that there's no reopen link (D6). **Retires ADR-0023 S3's in-app refine loop** as unwanted, not unscheduled (D7). **S1 is schema-free** (`AIHandoff` is device-local; `workbenchLog` already exists); **S2 adds three nullable synced `workbenchLog` columns** â€” `hypothesis`/`change`/`rationale` â€” because an experiment is **write-many** (its `outcome` is filled in later) and ADR-0040 puts human-repairable content in typed fields, **not** smeared into `body` (OQ2, resolved 2026-07-20: pre-prod is the reason to fix the shape *now*, not a licence to defer it; contrast ADR-0041's withdrawn `PlaybookSectionMeta`, which failed on *justification*, not cost). **Hand-validated end-to-end (OQ6, 2026-07-20): all slices un-gated.** With the v1 contract living only in the project's custom instructions, a long real conversation switched cleanly into the return block on `finalize`, echoed `YC-CONTRACT: v1`, and held all three field labels â€” confirming D1/D4/D8 in the shape we'd ship. **One binding failure: blank lines between blocks did not survive the paste path**, so the parser splits on the **label cycle**, never whitespace. **(D9)** threads carry a derived `<TaskType>: <Object>` title â€” task, not object-kind, since ADR-0041 section-scoping gives one recipe three live threads â€” and the title is **advisory only** (the auto-titler paraphrased it in testing; nothing parses it). Also **D8: the experiments verb emits no learnings** â€” 3 of 4 returned "learnings" were its own untested hypotheses restated as fact, which is structural (a conjecture establishes nothing), so knowledge is recorded later in the experiment's `outcome` â€” an experiment is a *pending* learning (Accepted 2026-07-20; S0/S1 dispatchable, S2 the schema slice behind them)
- [ADR-0032](ADR-0032-workbench-reference-material-fetch.md) â€” **Workbench reference material, fetched app-side.** Workbench chat context is a static serialization of already-stored data â€” the model never sees the **source page**, and the cook cannot point the discussion at a URL that isn't a candidate. The naive fix (put the URL in the prompt and tell the model to read it) is **rejected**: with no tool the model hallucinates plausible recipe content, the one unacceptable failure for a fidelity-first app. Each provider's **native server-side web tool** is also rejected as the mechanism â€” it means three mechanisms behind one flag (Anthropic tool / OpenAI tool / on-device none), it cannot reach **gated** sources (the Milk Street login wall, [[paywall-gating-taxonomy]]), and it hands the fidelity boundary to the provider; **doing the fetch ourselves is the LLM-agnostic implementation and the only one that solves gating.** So: a per-workbench **reference list** (URL â†’ fetched by our own engine (ADR-0007), **reduced** to readable text, cached CompareAlignment-style; or **pasted text** as the gating fallback), injected as **plain grounded text** behind the frontier budget, deduped against candidates, trimmed first on-device. Advisory **read** only â€” never grocery, pantry, or any persisted recipe field ([[llm-vs-determinism-surface-boundary]]); fetched web text is **data, never instructions**. Reuse not rebuild: fetch (ADR-0007) + reduce (existing editorial-prose extraction) + cache (ADR-0022 pattern) + inject (`WorkbenchChatContext`).
